# Propose Next Move

One action. The single most important thing to do next on any deal.

## When to Use
- User asks "what should I do next on [deal]?"
- Deal is stalled and needs a move
- After any deal update that changes the landscape

## Process

### Step 1: Load Deal State
Read deal memory. Identify:
- Current stage and {{methodology.framework}} scores
- Days since last activity
- Last action taken and outcome
- Weakest factor in the methodology

### Step 2: Classify Deal State

| State | Signals | Mode |
|---|---|---|
| **Advancing** | Recent activity, champion engaged, next step booked | Keep momentum |
| **Stalled** | 7+ days no activity, no next step | Create urgency |
| **At Risk** | Champion gone quiet, competitor surfaced, budget cut | Recover or exit |
| **Near Close** | EB engaged, paper process active | Remove friction |

### Step 3: Select Move

**Advancing**: Pick the move that advances the weakest methodology factor.
**Stalled**: Pick from recovery playbook:
- Re-engage champion with new value (not "just checking in")
- Go around: find a different thread into the org
- Compelling event: surface or create urgency
**At Risk**: Pick from triage:
- Verify: is the deal actually dead? Get a direct answer.
- Escalate: involve your manager or exec sponsor
- Exit: cut losses and reallocate time
**Near Close**: Remove friction:
- Pre-answer procurement questions
- Draft the MSA redlines
- Coach champion on internal approval pitch

### Step 4: Format the Move

## Output Format
```
NEXT MOVE: [Deal Name]

STATE: [Advancing / Stalled / At Risk / Near Close]
WEAKEST FACTOR: [methodology factor]

ACTION: [Specific action]
BY: [Date]
BECAUSE: [Why this matters now]
EXPECTED OUTCOME: [What changes if this works]
```

## Never Do
- Never recommend "follow up" without a specific angle
- Never recommend more than one action
- Never recommend an action in the Uncontrolled bucket
- Never recommend "schedule a call" when the problem is value, not access
